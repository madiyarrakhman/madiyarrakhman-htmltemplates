# Настройка Health Check в DigitalOcean

Судя по скриншоту, у вас настроен порт **3000**. Я обновил код, чтобы он тоже работал на порту 3000 по умолчанию.

Теперь нужно правильно настроить Health Check (проверку работоспособности), чтобы DigitalOcean знал, когда приложение запустилось.

### 1. Перейдите в настройки App Platform
1. Зайдите в **Settings** -> **Components** -> **[Ваше приложение]**.
2. Найдите секцию **Health Checks**.
3. Нажмите **Edit** напротив **Readiness check**.

### 2. Заполните поля следующим образом:

- **Type**: `HTTP`
- **Port**: `3000` (важно!)
- **Path**: `/api/health`
- **Initial Delay Seconds**: `15` (дайте приложению время на старт)
- **Period Seconds**: `10`
- **Timeout Seconds**: `5`
- **Failure Threshold**: `3`

### 3. Проверьте Run Command
В секции **Commands**:
- Убедитесь, что **HTTP Port** выставлен в `3000`.
- **Run Command** можно оставить пустым (тогда будет использоваться команда из Dockerfile) или оставить `npm start`, так как мы сделали для нее "заглушку".

### Почему это важно?
Ваш бэкенд имеет специальный удобный эндпоинт `/api/health`, который всегда возвращает статус 200, если сервер жив. DigitalOcean будет опрашивать его. Если он получит ответ, он переключит трафик на этот контейнер.
